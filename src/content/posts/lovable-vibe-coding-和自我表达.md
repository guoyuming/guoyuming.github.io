Lovable，是一个海外的 AI SaaS，让用户通过自然语言生成对话生成 web app，也就是一句话建站。用户主要来自欧美、印度和巴西。

![image.png](/images/lovable-vibe-coding-和自我表达/image.png)

看 Lovable 社区用户分享他们做的 web app，会觉得大部分都是玩具，怎么能用在生产环境呢？但它从一百万美金 ARR 到一亿美金 ARR，用了 8 个月。

![image.png](/images/lovable-vibe-coding-和自我表达/image%201.png)

Lovable 社区的 web app 可以二创。统计 100 个最多二创的 web app 模板：

| --- | --- | --- |

| toC 工具 | 21 | 股票行情看板、密码生成器、戒指尺寸换算、AA平摊计算、球赛计分器、记账软件 |

**官网**

做官网属于建站领域。不同于国内商户生意依赖平台，海外尤其是欧美，商户的生意围绕自己官网。官方呈现产品和服务、表达品牌主张、承接客户线索和服务预定。只要是个生意，就想做官网。海外各行各业的独立站可能至少几亿，有货架交易功能电商独立站是千万级别。

之前无代码建站用 Wordpress, Wix 这类工具。前端展示用页面模板，如果需要复杂功能如评论、地图，就需要另外装插件。虽然欧美用户普遍动手能力强，在 GUI 上的搜索、挑选、学习、调试，在画布和页面编辑器中拖拽的成本，还是会劝退一部分人。

更广泛和刚性的需求来自定制化。我的品牌官网，要和别人的都不一样，要有我自己的风格调性、以及复杂交互功能。无代码而想要定制，标准建站工具似乎不能满足。所以生态中出现了基于 Wordpress 的二次定制开发的外包团队和 Freelancer。

Lovable 社区分享大部分网站，偏设计平庸，功能简单。当然这部分是设计师 agent 能力不够，而非 代码 agent 能力不够。

但也出现了高质量的网站：带有交易、社区、双边平台功能，设计审美在线。（未从考证它是不是经过代码二次开发）。这个级别的网站，专业设计师能看出诸多瑕疵，但对中小商户已经够用。

**toC 工具和网页游戏**

如果说抖音极大降低了视频从创意到实现的门槛，带来了丰富内容的大爆发。Lovable 也有潜力降低想法的工具、游戏和网站的门槛，带来个性化、定制化工具的大爆发。

我们大部分人都有片刻起心动念，想到：要是有一个某某 App 帮我做某某事情就好了。

这些 App 本身不太具备商业价值，并且因为想法不成熟、太个性化，不会写代码、找外包成本太高，我们停止了这个想法。

但如果想法到 App 的成本是几美元呢？我们将看到个人专属的 App 涌现出来。

Lovable 社区已经有诸如习惯打卡、语言学习、冥想练习、密码生成器这类轻量工具。这类轻量工具，以前是独立开发者做的（记账、日记、Todo）。

**几个产品观点**

1、Wordpress 这类建站工具，做简单功能和交互、视觉要求不高的网站，已经比较好满足市场需求了。Lovable 切入的需求是：复杂功能、定制化、更个人使用的 web app，这部分需求过去是由外包公司和独立开发者。Lovable 用几美元、几天的成本，完成了过去价值几千美元，几周完成的定制开发任务。事实上已经有外包公司用 Lovable 给客户做应用赚钱了。

2、现在看起来很多很粗糙、简陋的交付，只是用户想法从 0 到 1 的实现出的 MVP 版本或原型。如同专业团队做产品不断迭代，Lovable 用户也能继续用自然语言做持续迭代。但不管是 V0 还是 V10 版本，每次迭代，门槛和成本都会显著低于雇人写代码。

3、Lovable 做出网站的好看程度，不取决于 Coding agent，更依赖设计师 agent。出众的、有风格和审美高级的网站设计总是少数，用来训练算法的大量前端 CSS 代码数据，也许都是普通设计样式，所以 AI 做出了符合最大公约数的网站设计。而用户也很难描述清楚，什么样的网站设计是好看的。这个特点在文本生成领域也是如此，不用 prompt engineering、few-shot、或者微调模型，AI 写不出有风格的文章。

3、Coding agent 代码能力很强，但自然语言不一定是和 AI agent 最高效协作的方式。软件开发是很复杂的协作，产品经理和设计师通过白板、设计草稿指指点点、当面沟通，还以来双方专业领域经验中大量隐性知识。Lovable 团队也深知这一点，提供了页面编辑器功能，用 UI 交互来修改页面样式。

3、Lovable 现在能做轻量级工具，往复杂企业管理软件（toB SaaS）做，它的天花板在哪里？Reddit 有用户说想用 Lovable 做 SaaS 赚钱，现在还没看到社区 toB SaaS 应用的优秀例子。toB SaaS 因为代码行数多、逻辑复杂，AI 智能需要提升，才能做出来。

4、Lovable 也不需要做所有企业管理软件，因为定制工作流的价值很难超过成熟企业管理软件带来的价值，在其背后是 SaaS 产品凝结了全行业多年的优秀实践。一个公司如果要做 CRM，那么用 Salesforce 和 Hubspot 不只是得到了提效工具，而是企业销售管理的漏斗模式（Sales Pipeline）理念，它把复杂多样的销售过程抽象成”线索到真实客户的转化阶段变化”。如果一家公司想用 AI 重新做一个 CRM，即使 Coding Agent 足够智能，用户也无法”说“出那么多有价值的需求。

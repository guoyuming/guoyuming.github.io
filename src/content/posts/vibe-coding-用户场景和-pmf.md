---
title: "Vibe coding 用户场景和 PMF"
---

一个公司产生商业价值的三个路径：1）更好的商品和服务，2）更高的交易撮合效率，3）更低的价格。

Vibe coding 是新领域，通过 Agent 软件和服务，给用户交付软件。在此之前，有建站工具、低代码平台、设计工具、Frontend Cloud 等垂直行业 SaaS 满足用户需求。

Vibe coding 的技术，有潜力统合各个垂直行业和碎片化工具，并泛化用户场景。

按大类分，用户有创业者、白领和创作者。

**创业者和资源瓶颈**

Vibe coding 产品顶流 Lovable 创始人 Anton 在一个[访谈](https://www.youtube.com/watch?v=DHLczPQj9rA)提到，他希望让 YC Founder 创业要的所有软件需求，在 Lovable 中都能实现。

当然，创业者也包括大量实体和服务业的“生意人”，他们在 Vibe coding 之前，需要雇佣研发团队或者用大量 SaaS 工具。研发团队在世界上大部分地方供不应求。

衡量产品价值，可以看创业者的业务，有多大比例依赖 Vibe coding 做出的软件：

- 理想态：创业者 100% 的业务依赖 vibe coding
- 其次：业务核心系统，比如官网建站和维护
- 最后：做 MVP、POC、原型，之后换其他工具 build

目前看到的例子，官网因为是需求高度结构化和标准化的商业场景，且实现难度在大模型能力范围内，跑出比较多的 use case。官网的进阶场景，是电商独立站，加上了营销和支付能力。

**白领的赋能提效**

大量实体行业和服务业公司，没有全职程序员或设计师。设计和代码技能有复杂度门槛。整个世界还在不断数字化和在线化，催生了大量设计稿、网页、后台工具、以及自动化需求。

这是欧美 SaaS 行业繁荣的原因。在大众设计领域有 400 亿美元估值的 Canva，工作流自动化有估值 25 亿美元的 n8n，以及 6500 亿美元的软件市场。

Vibe coding 在部门或个人工作生产力场景的价值是：Powerup 和省力

- Powerup 实现人难以完成，且标准化 SaaS 覆盖不到的个性化需求
    
    最近遇到一个真实案例是，朋友在互联网公司做运营，需要一个在线抽奖工具，规则要求某些账号有 1 次抽奖机会、有些账号可以抽多次。她搜索了一个晚上没有找到能满足需求的，打算找研发要资源定制实现。Manus 对话 10 分钟可以做出符合她需求抽奖工具。
    
- 省力在于，人可以完成的标准化需求，比传统工具交互成本低
    
    专业设计和 page builder 软件有学习曲线，夸张如 PS 或 Webflow，初学者第一次看到直接 overwhelm。非专业人士几乎很难有 mindset 和精力，在几十个菜单栏选项和几百个配置项中做出自己想要效果。前置一步是我们普通人也不具备专业设计的排版和审美知识。做 PPT 是简单的，但大部分人也不觉得它过程愉快。
    

**创作者的自我表达**

相比在创业和上班的商业场景形成了共识，各个公司都在卷，Vibe coding 给创作者的价值，目前行业普遍看空。因为轻量工具和小游戏，无法被分发，作品不被看见，创作者很难获得持续收入和社区反馈。

但这是我觉得最有趣的场景。

如果说抖音极大普及个人表达，让创作和表达从几千万的小众人群扩大到几亿。Vibe coding 也让大家实现自我表达。用抖音表达的用户，普遍有“镜头权力”，他们对持续出镜、露脸这件有热情。

通过一个软件作品来自我表达，是另外一类人群了。在 Reddit 上看到的例子：一个 Lovable 德国用户的作品，叫做 Bricklimit，给乐高收藏者的物料管理和数据分析工具。作者在 Reddit 为产品找种子用户，提到做这个 SaaS 初衷不是赚钱，而是 “build something useful, honest, and community driven”。

上述场景的 PMF 一直在变的原因，除了模型能力之外，我觉得有：

- 只用 Vibe coding 不足以满足用户需求，而工具生态还处于早期。
    - 所以我们看到动手能力强的用户，在用多个工具和各种 Hack 用法。Lovable 做出第一版页面后，微调页面元素，它的 Visual Edit 很难用，远不如拖拽所见即所得的 page builder。
    - AI 作为某项技术一骑绝尘，而服务于 AI 技术的应用和基础设施还在搭建。新技术的普及时间可能会很长：瓦特在 1769 年发明了蒸汽机。三十年后，英国大多数棉纺厂仍然依靠水车驱动，部分原因是运输煤炭用于蒸汽机很困难。直到 19 世纪初蒸汽动力铁路的发展，这种情况才有所改变。
- 用户本身的弹性需求和众多竞品
    - 创业者自身业务没找到 PMF 时候，他们需求也不固定，并且在多个竞品间来回流窜。白领需要做内部工具，他们发现 Manus 和 Gemini 也有这个功能。非全职的创作者，兴趣驱动，不会高频使用创作工具。
- 自然语言交互降低了工具门槛，产品泛化到了更广泛的人群。有部分人群没用过 SaaS，首个生产力工具是 Vibe coding，他们的画像、习惯、认知水平，从业者对他们了解还不够深。
